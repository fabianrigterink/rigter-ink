Operations research (OR) is the discipline of applying mathematical methods to decision-making. If you've ever wondered how airlines schedule crews, how delivery routes are optimized, or how warehouses manage inventory — that's OR at work.

## Linear programming

The workhorse of OR is **linear programming** (LP). An LP problem minimizes (or maximizes) a linear objective subject to linear constraints:

$$
\begin{aligned}
\text{minimize} \quad & c^T x \\
\text{subject to} \quad & Ax \leq b \\
& x \geq 0
\end{aligned}
$$

Despite its simplicity, LP can model a surprising range of real problems. Here's a production planning example using PuLP:

```python
from pulp import LpProblem, LpVariable, LpMaximize

# A factory makes chairs and tables
prob = LpProblem("production_planning", LpMaximize)

chairs = LpVariable("chairs", lowBound=0, cat="Integer")
tables = LpVariable("tables", lowBound=0, cat="Integer")

# Maximize profit: $45 per chair, $80 per table
prob += 45 * chairs + 80 * tables, "Total Profit"

# Constraints: wood and labor hours available
prob += 5 * chairs + 20 * tables <= 400, "Wood"
prob += 10 * chairs + 15 * tables <= 450, "Labor"
prob += chairs <= 40, "Chair Demand"

prob.solve()
print(f"Chairs: {chairs.value()}, Tables: {tables.value()}")
print(f"Max profit: ${prob.objective.value()}")
```

## The traveling salesman problem

The TSP is perhaps the most famous combinatorial optimization problem. Given $n$ cities, find the shortest route that visits each city exactly once and returns to the start.

The number of possible routes grows factorially — for 20 cities, there are over $10^{17}$ possibilities. Exact algorithms can handle modest sizes, but for larger instances, we turn to heuristics and metaheuristics.

## Constraint programming

When your problem has complex logical constraints, constraint programming (CP) can be more natural than LP. CP excels at scheduling, timetabling, and configuration problems.

```python
from ortools.sat.python import cp_model

model = cp_model.CpModel()

# Schedule 5 tasks on 2 machines, minimizing makespan
num_tasks, num_machines = 5, 2
durations = [3, 2, 4, 1, 3]
horizon = sum(durations)

# Decision variables
starts = [model.NewIntVar(0, horizon, f"start_{i}") for i in range(num_tasks)]
ends = [model.NewIntVar(0, horizon, f"end_{i}") for i in range(num_tasks)]
machine = [model.NewIntVar(0, num_machines - 1, f"machine_{i}") for i in range(num_tasks)]
intervals = [model.NewIntervalVar(starts[i], durations[i], ends[i], f"interval_{i}")
             for i in range(num_tasks)]

# Objective: minimize makespan
makespan = model.NewIntVar(0, horizon, "makespan")
model.AddMaxEquality(makespan, ends)
model.Minimize(makespan)

solver = cp_model.CpSolver()
status = solver.Solve(model)
if status == cp_model.OPTIMAL:
    for i in range(num_tasks):
        print(f"Task {i}: machine={solver.Value(machine[i])}, "
              f"start={solver.Value(starts[i])}, end={solver.Value(ends[i])}")
```

## When to reach for OR

Not every problem needs a sophisticated optimization model. But if you're making decisions that involve trade-offs between competing objectives and constraints, OR gives you a principled framework for finding the best possible solution — or at least proving how close to optimal your heuristic gets.
