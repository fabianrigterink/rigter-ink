## Self-attention: the core mechanism

The fundamental operation in a transformer is **scaled dot-product attention**. Given queries $Q$, keys $K$, and values $V$, attention is computed as:

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

where $d_k$ is the dimension of the key vectors. The scaling factor $\frac{1}{\sqrt{d_k}}$ prevents the dot products from growing too large, which would push the softmax into regions with vanishing gradients.

## Why does this work?

Think of it as a soft dictionary lookup. Each query asks "what information do I need?", the keys say "here's what I contain", and the dot product measures relevance. The softmax converts these relevance scores into a probability distribution, and the values provide the actual content to aggregate.

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    """Compute scaled dot-product attention."""
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)

    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))

    weights = F.softmax(scores, dim=-1)
    return torch.matmul(weights, V), weights
```

## Multi-head attention

Rather than computing a single attention function, transformers use **multi-head attention** — running $h$ attention functions in parallel, each with different learned projections:

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O
$$

where each head is:

$$
\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)
$$

This allows the model to attend to information from different representation subspaces at different positions. One head might learn syntactic relationships while another captures semantic similarity.

## Positional encoding

Since attention is permutation-invariant — it doesn't inherently know about token order — transformers inject positional information using sinusoidal functions:

$$
PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right)
$$

$$
PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)
$$

The intuition: each dimension of the positional encoding corresponds to a sinusoid with a different wavelength, forming a kind of binary encoding of position that the model can learn to decode.

## The feed-forward network

Between attention layers, each position passes through an identical feed-forward network:

$$
\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2
$$

Recent work has shown this component acts as a key-value memory, storing factual associations learned during training. The first linear layer projects to a higher dimension (typically $4 \times d_{\text{model}}$), and the second projects back down.

## Putting it all together

A transformer layer combines multi-head attention and feed-forward networks with residual connections and layer normalization:

$$
x' = \text{LayerNorm}(x + \text{MultiHead}(x, x, x))
$$

$$
\text{output} = \text{LayerNorm}(x' + \text{FFN}(x'))
$$

Stack $N$ of these layers, and you get the remarkable capability we see in modern language models. The depth allows the network to build increasingly abstract representations — from tokens to syntax to semantics to reasoning.
