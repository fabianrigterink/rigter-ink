Coal from multiple mines in the Hunter Valley travels through a shared rail and port network before being loaded onto ships. The system must be scheduled so that trains, stockpiles, and ship loaders operate efficiently and each ship receives coal meeting its quality specification (e.g., energy content, ash percentage).

We model this as a discrete flow pooling problem — a variant of the pooling problem where flow quantities must be integer multiples of a base unit (train load sizes are discrete). We show that this discrete structure, while initially seeming to make the problem harder, actually enables exact solution methods that would not apply to the continuous case. We formulate the problem, prove complexity results, and demonstrate computational results on instances derived from the actual Hunter Valley network.
