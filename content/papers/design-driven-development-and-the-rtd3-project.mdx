Design Driven Development (DDD) is a software development methodology that treats design — the structure and organisation of a system — as a first-class concern, rather than something that emerges from implementation or is captured only in diagrams. The core idea is that design decisions should be made explicitly and early, and that code should be structured to reflect and enforce those decisions.

This thesis introduces DDD and presents the rtd3 project, a prototype framework that implements DDD principles. The framework provides tools for defining design contracts, detecting violations at development time, and visualising the structure of a system. The goal is to make the gap between intended design and actual implementation visible and actionable.
