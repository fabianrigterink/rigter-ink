The pooling problem has two classical formulations: the $p$-formulation (variables track the fraction of flow from each source in each pool) and the $q$-formulation (variables track total flow on each arc). Both yield LP relaxations that can be far from the true optimal — the integrality gap is large.

We introduce a new family of multi-commodity flow (MCF) formulations. Instead of tracking aggregated quality fractions, MCF formulations track how much of each source's material reaches each output pool or sink, via each possible path through the network. This introduces more variables but yields significantly stronger LP relaxations. We prove that the MCF relaxation dominates the $p$- and $q$-relaxations, derive conditions for when it is exact, and demonstrate substantial improvements in computation times on benchmark pooling instances.
